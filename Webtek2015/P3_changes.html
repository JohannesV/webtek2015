IT2805 November 11th 2015

Changes from P1 to P2

We have made some changes throughout the entire project. From phase one(Research P1) to phase two(Design, P2) we have made some changes to the content of the web site.

  • Changed "Contact" to "Information":
    We have changed the name of the contact page to information. We chose to change the name of this page so that the user would know that there is more information to be found than solely contact information. In our opinion, this new title express more clearly the content within the page.

  • Added opening hours:
    We have added opening hours to the information page. This is an important feature that we forgot to mention in the P1 part of the project. Opening hours is essential information that the customer would expect to see. This in order to plan the booking of an appointment, and to know when you should expect to be able to get in contact with the salon.

  • Added slide show:
    In the design phase, P2, we set out to define more clearly how to execute our statement for the home page from phase P1; “The main page is intended to catch the attention of potential customers (...)”. In order to achieve this, we added a slide show to the home page in order to catch the attention of the users, as was the criteria from the start.

  • Removed "Hair Salons":
    When we worked with the design of the page, we chose to get rid of the hair salon page. The reason for this is that we wanted to work with a high end, Trondheim-based salon in stead of a chain. There is no longer any need for a distinct page where you display different salons in a chain, because we are not working with a chain with several locations.

  • Added gallery page:
    We have decided to add a gallery page. The gallery will be accessed from the navigation bar, and replace the hair salons site. A gallery page will let the hair salon show their work to their customers. In addition, a gallery page will hopefully make the web site more interesting for the customer, adding more visuality of the work done by the salon alongside the description of the different hairdressers specialities in the hairdressers view.

Changes from P2 to P3

We encountered some challenges realizing the design defined in phase two(P2). This has resulted in some changes from our initial idea of which methods we should use for the coding. There has also been added some new features to the web site.

  • Use mapbox instead of leaflet:
    In P2 we set out to implement a map function in the information page using leaflet. In coding the map function however, we ended up using mapbox instead of leaflet in order to implement the feature. This was mainly due to few and poor tutorials considering leaflet. Because neither leaflet or mapbox has been covered in the course, we were dependent of good tutorials in order to understand and use it. The js for the map function is located in the file information.html.

  • Use of html forms:
    The booking page described in the project design, P2, was a very ambitious booking site with a vast amount of styling. We wanted to implement this type of booking page, but eventually came to the realization that we did not have the time to implement it.

    Therefore, we had to come up with an alternative booking page, and we chose to use html forms for this. This decision was based on the fact that we have learned about this type of form during the course. In addition, html forms will be a good alternative for the booking page, because it gives us the opportunity to validate the information from the user in order to ensure that the user books an appointment right. The information will not be sent anywhere because of the fact that we do not have an actual client. We also do not have a system in place for processing the information collected by the form. Forms validation is not being handled with javascript, but with CSS and HTML5. The functions HTML5 provide was sufficient for our needs.

  • Addition to navigational structure:
    We have added an extra layer to the navigation structure. On all pages, clicking the logo will lead the user back to the home page. This is a standard that is widely used in web page design, and we have chosen to implement this as it will be something that is easy to do, and because it will enhance the user experience for the users that expect it to be there. It will be a subtle feature that meet some user expectations and that does not disturb the users that chose not to use it.

  • Changed logo:
    In order to give the web page a stronger identity, we have come up with a logo for the fictive hair salon. The logo is displayed above. It is a circle with black stroke. Inside the circle, the generic name of the hair salon “The Hair Company” is written. Wrapped around the lower inside of the circle, it says “Since 1986”, which we have set as the start-up year of the company. Above the company name, there is an image of scissors, which together with the name indicates that the company is a hair salon.
